---Базовые теги HTML---

<!DOCTYPE html> - Сообщаем браузеру как стоит обрабатывать эту страницу (т.е. в формате HTML)

<html lang="ru">...</html>- Оболочка документа, указываем язык содержимого

<head>...</head>- Заголовок страницы, контейнер для других важных данных (не отображается)

<title>...</title>- Заголовок страницы в браузере

<link rel="stylesheet" href="css/style.css">- Привязка CSS файла к HTML

<meta http-equiv="Content-type" content="text/html; charset=UTF-8">- Кодировка страницы


Теги «title» и «meta» - содержатся внутри тега «head», а сам «head» внутри тега «html». После тега «head» следует тег «body»


<body>...</body>- Отображаемое тело страницы

<header>...</header> и <footer></footer>- блочные теги (на всю длину страницы). Отображаются вверху/снизу тела страницы

<h1..h6>...</h1..h6>- заголовки разных размеров

<div>...</div>- блочный тег, блок информации (всегда с новой строки и занимает всю длину)

<span>...</span>- строчный тег, блок информации (равен длине текста)

<p>- блочный тег, создает новый параграф. Используется для разграничения абзацев. Браузеры по умолчанию добавляют отступы до <p> и после </p>

<img src="/source/pics/cat.jpeg" alt="Aleq" width="200" height="185" />- используется для вставки изображений. Атрибут «alt» служит для отображения текста вместо картинки, если картинка по каким-то причинам не загрузилась. Также можно указать размеры с помощью атрибутов «width» и «height»

<a href="www.google.com" target="blank">Google</a>- создает ссылку на страницу в интернете. При указании атрибута target="blank", сайт откроется в новой вкладке. В качестве ссылки можно использовать как текст, так и изображение.


<p id="top">Оглавление</p>- использование якорей, для перемещения по странице.

<a href="#top">Наверх</a>- якорь работает по атрибуту «id».


<br>- перенос на новую строку

<hr>- горизонтальная линия

Теги списков

<ul>- маркированный список (Unordered List). Имеет несколько типов маркировки: type="disk/circle/square"

  • disk
  • circle
  • square

<ol>- нумерованный список (Ordered List) Имеет несколько типов маркировки: type="1" - цифры , type="А" - заглавные буквы, type="а" - строчные буквы, type="I" - заглавные римские цифры, type="i" - строчные римские цифры. Атрибут start="5" - c какого значения начать отсчет. Атрибут "reversed" - нумерация в обратном порядке

  1. цифры
  2. заглавные буквы
  3. строчные буквы
  4. заглавные римские цифры
  5. строчные римские цифры

<li>- элемент списка (List item)


<dl>- список определений (Definition List)

Имеет 2 сопутствующих тега:

<dt>- термин в списке определений (Definition Term)

<dd>- определение предшествующего термина в списке определений (Definition Details)

Теги таблиц

<table>- основной тег, контейнер для строк и ячеек таблицы

<tr>- строка в таблице (Table Row). Содержит внутри себя ячейки таблицы

<td>- ячейка в таблице (Table Data). Ячейки в таблице образуют столбцы, т.к. отдельного тега для столбцов в HTML нет

<th>- тег для заголовков ячеек. По умолчанию выделяет текст жирным и центрирует его в ячейке

---Теги форм---

<form action=""></form>- оболочка формы (контейнер), блочный тег, отвечает за передачу данных. В графе «action» указывается путь. Оставив это поле пустым - будет ошибка. Можно поставить заглушку #.

<form method=""></form>- метод передачи формы. Атрибут «method» может иметь 2 значения:

«get» - используется по умолчанию. Передача данных идет прямо в адресной строке, видны пользователю и могут быть использованы неоднократно.

«post» - передает данные скрытно используя запрос браузера. Подходит для больших объемов информации. Может передавать прикрепленные файлы с атрибутом «enctype="multipart/form-data"»

Разновидности тега «input»

<input type="text" name="username">

Создает форму для ввода текста, например, имени пользователя. Вводимый текст виден пользователю. Значение «value» впишет в форму определенный текст заранее.



<input type="password" name="userpass">

Создает форму для ввода пароля. Вводимые данные не видны пользователю.



<input type="tel" name="userphone" value="+">

Создает форму для ввода номера телефона (для мобильных устройств включится спец.клавиатура)



Ко всем типам форм с вводом данных можно применить атрибут «placeholder=""», который напишет подсказку внутри поля.


<label for="id_x">Выберите пол:</label>- используется для создания подписи к элементу формы, а также для расширения поля выбора/клика для "input" (особенно полезно для "radio" и "checkbox")

<fieldset><legend>Персональные данные</legend></fieldset>

Группирует элементы формы (поля ввода input, textarea, выпадающие списки select и другие) в блок с характерным выделением границ. Опционально с помощью тега legend внутри fieldset можно задать заголовок для создаваемой группы (он может быть только один и обязательно должен идти первым вложенным элементом).

<p><input checked type="radio" name="sex" value="male">Мужской</p>

<p><input type="radio" name="sex" value="female">Женский</p>

Форма для выбора в формате «или/или». Единственная форма, где должно быть указано одинаковое имя в атрибуте «name» (для общей группы радиокнопок). Можно перед одним из атрибутов «type» указать атрибут «checked», тогда один из выборов будет выбран заранее, при загрузке/обновлении страницы.

Персональные данные

<p><input type="checkbox" name="car" value="yes">Наличие авто</p>

Форма множественного выбора (в виде галочек). Также работает с атрибутом «checked» (ставит галочки заранее, при загрузке/обновлении страницы)


<p><input type="file" name="file">Скан/фото паспорта</p>

Форма прикрепления файла. Работает только в теге формы с атрибутом «post» и «enctype="multipart/form-data"» (см.выше).

Скан/фото паспорта:


Значения атрибута «name» всегда должны различаться во всех формах (кроме «radio»).

Другие теги форм

<textarea name="text" id="" cols="30" rows="10"></textarea>

Форма для ввода многострочного текста, например примечания или комментария. Атрибуты «cols» и «rows» можно удалять, так как размеры настраиваются позже с помощью JS и CSS

<select name="color">
<option value="red">Красный</option>
<option selected value="blue">Синий</option>
<option value="green">Зеленый</option>
</select>

Выпадающий список. Теги select и option всегда используются в сочетании друг с другом, причем тег option должен быть вложен внутрь тега select. Можно перед одним из атрибутов «value» указать атрибут «selected», тогда это значение будет выбрано заранее при загрузке/обновлении страницы. Если тегу select добавить атрибут «multiple», то форма перестанет быть выпадающей и станет в виде списка.

К любому тегу формы кроме form можно применить атрибут «disabled», который сделает форму неактивной и не отправит данные из неё на сервер.


Ко всем тегам можно применить атрибут «tabindex» с порядковым номером. Данный номер определяет очередность форм при переключении по ним с помощью кнопки «Tab». Для всех значений тега «input type="radio"» атрибут «tabindex» должен быть указан с одинаковым номером.


<button type="submit/reset/button" ></button>

Атрибут «type» задает поведение кнопки. Значение «submit» - отправляет данные на сервер (по умолчанию). Значение «reset» - удаляет введённые данные из формы. Значение «button» - просто кнопка. Действие для неё можно задать через скрипты.



<input type="date">- ввод даты без времени: год, месяц и дата.



Атрибут required сделает любое поле input, select или textarea обязательным для заполнения перед отправкой формы
